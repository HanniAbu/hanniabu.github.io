Verticulture Farms (Software)

*Due to contracts I have signed, I am unable to show pictures, screenshots, or code from the software.

Background:

Verticulture Farms is an aquaponics farming company. In aquaponics, there are various parameters that are important to the health of the plants and the fish. Solutions that currently exist are directed towards hydroponic farms, aren't easily extensible, and/or don't include data visualization.

Goal:

Create a system that will collect data, present it on a dashboard to view current and historical values, allow users to control farm operations, and automatically to adjust parameters that are out of range. The interface should be simple, elegant, and easy to use for non-technical people.

Design:

We collect data using sensors which are hooked up to a headless Raspberry Pi. The Raspberry Pi was modified to boot from the micro SD card and then run the file system off a USB drive. I made this decision for a few reasons. If there is a power failure, the flash drive has less of a chance of being corrupted, larger capacity flash drives are cheaper than equal capacity micro SD cards, running rsync is safer with this setup(if I remember correctly), and it's easier to move the flash drive from computer to computer if anything needs to be done with it. After retrieving sensor readings, the raw data is saved to a file. This file is then sent by LAN through SSH file transfer to our local desktop. I chose not to send anything over wifi to make the system slightly more secure. Once the desktop receives the file, the raw data is added to one MySQL database and it is also processed to put in the proper format and make sure the data is valid. I chose not to process the data on the Raspberry Pi in order to keep it lightweight. All the data, raw and processed, is stored in MySQL with 3NF normalization, and is periodically backed up and put onto an external HHD. MySQL was chosen since all the data is time-series. This data is then presented on a dashboard. The backend was built using PHP while the front-end was built using Bootstrap and jQuery. The dashboard presents the various sensors' current readings as a value, historical readings in graph form using chart.js, and a control panel. The control panel has options that allow the user to turn on/off lights, pumps, feeders, dosers, heaters, and fans, which are actuated through the Raspberry Pi. Incoming data from the sensors will be compared to recent readings and desired levels to check if the values are within range and/or heading out of range. If a parameter is going out of range, it will be automatically adjusted accordingly to bring the values back to acceptable levels.
